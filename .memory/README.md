# Project Memory

This directory stores persistent operational memory for the master's thesis repository.

Read **`.cursor/agents/hypo-stage-masters-thesis-agent-instructions.md`** first for the full agent workflow, then use the files below when editing the thesis.

Use these files before making changes:

- `thesis-project.md`: canonical thesis identity, scope, repo conventions.
- `ime-workflow.md`: IME-USP workflow, deadlines, and post-defense steps.
- `defense-and-committee.md`: defense schedule and committee composition.
- `writing-conventions.md`: citation, wording, and formatting conventions learned during revisions.
- `session-log.md`: append-only summary of important repository-level decisions.
- `knowledge-sources.md`: curated list of repositories, PDFs, institutional pages, academic references, and gray-literature sources to consult when project memory is not enough (read after the other `.memory/` files when you need deeper context).

Rules:

1. Always read `.memory/` before editing the thesis.
2. If more context is still needed, consult `knowledge-sources.md` before broad searching.
3. When institutional facts change, update both `.memory/` and any relevant LaTeX/config files.
4. Do not treat `.memory/` as publishable thesis content.
